# AI Mock Interview Platform

An AI-driven mock interview platform designed to help developers prepare for technical interviews. The platform analyzes candidate resumes, tailors questions based on selected roles and difficulty, transcribes spoken answers, supports coding exercises with an embedded Monaco editor, and provides detailed AI-powered feedback.

---

## 🚀 Key Features

*   **Custom Interview Setup:** Pick a target role, set the difficulty level (Easy, Medium, Hard), and upload a PDF resume.
*   **Resume Analysis:** Automatically parses and evaluates PDF resumes to generate personalized, context-aware interview questions.
*   **Interactive AI Interviewer:** A conversational interface that walks the candidate through questions using speech-to-text transcription.
*   **Monaco Code Editor:** Integrated code editor allowing candidates to answer technical programming questions directly in the browser.
*   **Detailed Analytics & Feedback:** Comprehensive scoring breakdown (overall score, technical knowledge, communication, coding skills) generated by Gemini, with tips for improvement.
*   **Interview History:** Tracks past performance over time so users can visualize progress.

---

## 🛠️ Tech Stack

### Frontend (Client)
*   **React 19** & **Vite**
*   **Monaco Editor (`@monaco-editor/react`)** for code-based questions
*   **React Router DOM** for navigation
*   **React Icons** & **React Hot Toast** for a modern user interface and notifications
*   **Axios** for API requests

### Backend (Server)
*   **Express.js** & **Node.js**
*   **MongoDB Atlas** via **Mongoose** for data storage
*   **Gemini API (`@google/genai`)** for question generation, resume parsing, and feedback
*   **AssemblyAI** for voice answer transcription
*   **Murf API** for text-to-speech voice generation
*   **Multer** & **pdfjs-dist** for secure PDF upload and text extraction

## ⚙️ Getting Started

### Prerequisites
Make sure you have [Node.js](https://nodejs.org/) installed.

### 1. Clone & Set Up Repositories

#### Setup Backend Server:
1. Navigate to the `server` directory:
   ```bash
   cd server
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Create a `.env` file in the `server` directory and add your environment variables:
   ```env
   PORT=5000
   NODE_ENV=development
   MONGODB_URI=your_mongodb_uri
   JWT_SECRET=your_jwt_secret
   JWT_EXPIRES_IN=7d
   GEMINI_API_KEY=your_gemini_api_key
   MURF_API_KEY=your_murf_api_key
   ASSEMBLYAI_API_KEY=your_assemblyai_api_key
   CLIENT_URL=http://localhost:5173
   ```

#### Setup Frontend Client:
1. Navigate to the `client` directory:
   ```bash
   cd ../client
   ```
2. Install dependencies:
   ```bash
   npm install
   ```

---

## 🏃 Running the Application

### Start the Backend Server
From the `server` directory:
```bash
npm run dev
```
The server will run on `http://localhost:5000`.

### Start the Frontend Client
From the `client` directory:
```bash
npm run dev
```
The client will run on `http://localhost:5173`.

## 📂 Project Structure

```
├── client/                 # React Frontend application
│   ├── src/
│   │   ├── components/     # Reusable UI elements (Navbar, ProtectedRoute, CodeEditor, VoiceRecorder)
│   │   ├── context/        # Auth Context provider
│   │   ├── pages/          # Login, Home, Setup, Interview, Feedback, and History pages
│   │   ├── services/       # Client-side API request functions
│   │   └── main.jsx        # App entry point
│   └── vite.config.js      # Vite configuration
│
└── server/                 # Express Backend application
    ├── src/
    │   ├── config/         # MongoDB, Gemini configuration
    │   ├── controllers/    # API controllers (auth, interview, resume, history)
    │   ├── middleware/     # Auth, upload, and error handling middlewares
    │   ├── models/         # MongoDB collection models (User, Interview, Resume)
    │   ├── routes/         # Express routing definitions
    │   └── services/       # Integration logic (Gemini, AssemblyAI, Murf)
    └── server.js           # Server startup script
```
